Industrial Data Platform — 7 Stappen

Je fabriek heeft geen dataprobleem.

Het heeft een contextprobleem.

Elke PLC publiceert data. Maar data zonder context is ruis. Wat jouw historian iedere seconde ontvangt, is dit:

Raw sensordata — wat jouw systeem ontvangt:

TIC-001 = 67.3
PIC-001 = 4.47
FIC-001 = 248.1
BatchCounter = 142 ← Wat betekent dit? Niemand weet het. Dit is het probleem.

De waarde 67.3 is nutteloos zonder te weten: van welke machine? In welke engineering unit? Wat is normaal? Wat is een alarm? Context is het stuk dat AVEVA je probeert te verkopen voor €40.000 per jaar.

De Data → Information ladder

DATA
TIC-001 = 67.3
↓ Stap 3: MODEL (ISA-95 context toevoegen)
INFORMATION
WC-PAST-01 · Temperatuur · 67.3°C · Normaal: 55–75°C
↓ Stap 4–5: STORE + ORCHESTRATE
INSIGHT
Temperatuur stijgt richting alarm · trend +2°C/uur
↓ Stap 6–7: VISUALIZE + DISTRIBUTE
ACTION
Operator past koeling aan · N8N stuurt Teams notificatie
De 7 stappen in deze gids leiden je langs elke trede van de ladder — van raw tag tot actionable information.

Industrial Data Platform — 7 Stappen

De 7 stappen

1
Stap 1

CONNECT — Data binnenhalen

Koppel aan de databron van de PLC of SCADA. OPC-UA is de industriestandaard: elk modern systeem spreekt het. MonsterMQ fungeert als OPC-UA client én MQTT broker in één container — geen aparte bridge nodig.

OPC-UA (asyncua) MonsterMQ MQTT Modbus (toekomstig)
2
Stap 2

CONDITION — Data betrouwbaar maken

Verwijder dubbele waarden, filter ruis, converteer engineering units. Een PLC publiceert 1x per seconde — maar niet elke waarde is anders. Alleen gewijzigde waarden opslaan bespaart 60–80% opslagruimte.

MonsterMQ flow engine N8N validatie
3
Stap 3 — KRITIEK PUNT

MODEL — Context toevoegen (DATA → INFORMATION)

Dit is waar data information wordt. Gebruik ISA-95 hiërarchie (Enterprise → Site → Area → WorkCenter → Tag) om elke sensorwaarde te contextualiseren. Een JSON model beschrijft elke machine: wat het doet, wat normaal is, welke alarms er zijn, en hoe het zich verhoudt tot andere machines.

Walker's vier namespace typen: Definitional (wat is het?), Functional (wat doet het?), Informative (wat meet het?), Ad-hoc (tijdelijk/batch).

ISA-95 JSON model UNS (Unified Namespace) MQTT topic structuur
4
Stap 4

STORE — Information opslaan

MongoDB slaat alle MQTT berichten op in de plc_data collectie met automatische timestamp. Retentie 30 dagen. Geen InfluxDB nodig — MongoDB handelt gemengde data (sensorwaarden, batch records, alarms, shift logs) in één collectie.

MongoDB MonsterMQ archivering 30-dag retentie

Stap 5

ORCHESTRATE — Information laten stromen

N8N automatiseert fabrieksoperaties: productieorders starten, alarm-notificaties naar Teams sturen, barcode-scans verwerken, shift rapporten mailen. In de IDP stack zijn 6 workflows gedefinieerd voor een complete packaging lijn.

N8N workflows MonsterMQ webhooks Teams / email
6
Stap 6

VISUALIZE — Insights zichtbaar maken

Grafana voor time-series analyse en ad-hoc queries (via FastAPI JSON datasource). Next.js voor live dashboards die elke 5 seconden refreshen — ideaal voor operators op de werkvloer. Grafana template variables: één dashboard voor 50 machines.

Grafana Next.js FastAPI
7
Stap 7

DISTRIBUTE — Action mogelijk maken

Het Unified Namespace (UNS) maakt data beschikbaar voor elk systeem dat het nodig heeft — ERP, MES, cloud, edge. MonsterMQ op een Raspberry Pi stuurt lokale data door naar de VPS. REST API voor externe integraties. MQTT retain flag voor startup states.

UNS / MQTT REST API Raspberry Pi edge
Totale kosten open source stack: €8/maand VPS · Geen Ignition · Geen HighByte · Nul extra licentiekosten.
Vergelijk: AVEVA Connect €40.000/jaar · Ignition Gateway €5.000/jaar + HiveMQ €15.000/jaar.
