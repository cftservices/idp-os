The Spotlight Q&A

What are you building, and who's it for?
I'm building an online education program for engineers who work with industrial machines — think production lines, packaging plants, food & beverage factories. These engineers deal with massive amounts of data from those machines every day, but most of that data is useless without context.

Right now, companies solve this by buying expensive software platforms — we're talking €30,000–€40,000 per year. My program teaches engineers how to get the exact same result using free, open-source tools, all running on a server that costs €8 per month.

How'd you land on that niche?
I didn't research it — I lived it. 15 years programming factory machines. I watched the same pattern repeat in every plant: brilliant engineers who understood the equipment inside-out, completely trapped by expensive vendor software just to make sense of their own data.

Then I built my own system using open-source tools for €8/month and it worked exactly as well as the €40K platforms. That was the moment. This isn't a market I found — it's a problem I had.
💡 "The expensive platforms don't sell you better data. They sell you context. I figured out how to build that context yourself."

What's working right now, and what's been the hardest part?
What's working: a daily system that moves every part of the funnel at once.

Every morning I run a tool that finds engineers on LinkedIn who deal with exactly this problem — I look at their profile, see their background, and send them a personal message. Not a pitch. A genuine, technical conversation opener. The ones who respond, I invite to a free Facebook community I'm building around this topic.

In that community I post three times a week — practical content about how to build with open-source tools. Meanwhile I'm making YouTube videos that teach the same framework, which also drive people into the community.

The goal is a full self-reinforcing loop: LinkedIn → Community → YouTube → Workshop → Course. Every day I push all five channels forward simultaneously.

Hardest part: the engineers I'm targeting are not impulse buyers. They're methodical by nature. They need to see you 7–11 times before they trust you enough to pay for anything. Building that trust system without burning out is the real challenge.

One thing you learned that you wish you knew on day one
Don't build the course first. I spent months building the platform — the website, the checkout, the whole online school infrastructure — before I had a single paying student.

What I wish I'd done: start with the conversations. Send the DMs. Build the community. Run one live workshop on a video call. Then package it into a course once you know exactly what people need to hear. The product isn't the hard part. Finding out what the product should be is.
Where are you headed next?
Getting the Facebook community to 50 members — that's the threshold to run the first live workshop. From there: run it live 2–3 times, refine the content, then lock it into an evergreen program that runs without me needing to be present.

The end goal is a fully automated education business: engineers find me through YouTube, join the free community, attend a workshop, and enrol in the course. I stay focused on making the content better. The system handles the rest.

The 7-Step Framework — Industrial Data Platform

The 7-Step Framework

How to go from raw machine sensor data to actionable information — using only open-source tools that cost €8/month to run.

1
Step 1
CONNECT — Get the data in
Connect to the machine's data source. Think of this as plugging a translator between the factory machine and your data system — the machine speaks one language, your platform speaks another. One piece of software handles both sides, no separate bridge needed.
OPC-UAMonsterMQMQTT
2
Step 2
CONDITION — Make the data reliable
Machines report data every second — but most of it is identical to the second before. Remove duplicates, filter out noise, only store what actually changed. Result: 60–80% less storage, and data you can actually trust.
N8N validationFlow engine
3
Step 3 — Critical Point
MODEL — Turn data into information
This is the step that changes everything. Raw data says "67.3". Information says "Temperature sensor on Pasteuriser Line 1 — reading 67.3°C — normal range is 55–75°C — currently OK." You add the context that makes the number mean something. This is what the expensive platforms charge €30K/year to do. You do it yourself with a JSON file.
ISA-95 modelUnified NamespaceContext layer
4
Step 4
STORE — Save the information
Store everything with a timestamp. One database handles all data types — sensor readings, batch records, alarms, shift logs — no need for separate specialist databases. Retention: 30 days by default.
MongoDB30-day retention
5
Step 5
ORCHESTRATE — Make information flow
Automate factory operations: start a production order, send an alarm to Teams, process a barcode scan, email a shift report. All without writing custom code — visual workflow builder. Six pre-built workflows cover a complete production line.
N8N workflowsTeams / email
6
Step 6
VISUALIZE — Make insights visible
Build dashboards that refresh every 5 seconds, trend charts for analysis, operator screens for the shop floor. One dashboard template covers 50 machines. Engineers actually like using these — because they built them.
GrafanaNext.jsFastAPI
7
Step 7
DISTRIBUTE — Enable action
Make your data available to every system that needs it — ERP, MES, cloud platforms, edge devices. Push data from a €35 Raspberry Pi on the factory floor to a cloud server. REST API for any external integration. You own every connection.
